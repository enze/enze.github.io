第一章 程序设计和C语言

发表于 2018-04-01 | 分类于 C

1.1 什么是计算机程序

  • 程序：一组计算机能识别和执行的指令
  • 只要让计算机执行这个程序，计算机就会自动地、有条不紊地进行工作
  • 计算机的一切操作都是有程序控制的，离开程序，计算机将一事无成

1.2 什么是计算机语言

  • 计算机语言：人和计算机交流信息的、计算机和人都能识别的语言
  • 计算机语言发展阶段
    • 机器语言 (由0和1组成的指令)
    • 符号语言 (用英文字母和数字表示指令)
    • 高级语言 (接近于人的自然语言和数学语言)
    • 面向过程的语言 (非结构化的语言、结构化语言)
    • 面向对象的语言

1.3 C语言的发展及其特点

  • C语言是国际上广泛流行的计算机高级语言
  • C语言的发展
    • BCPL
    • B
    • C
  • 标准 C99
  • 用途广泛、功能强大、使用灵活的过程行编程语言，既可以编写应用软件，又能编写系统软件。
  • 主要特点
    • 语言简洁、紧凑，使用方便、灵活

      只有37个关键字
      9种控制语句

    • 运算符丰富

      34种运算符
      括号、赋值、类型转换
      表达式类型多样化

    • 数据类型丰富

      整型、浮点型、字符型、数组类型、指针类型、结构体类型、共用体类型
      C99 复数服点类型、超长整形、布尔类型
      指针类型数据，能用来实现各种复杂的数据结构 (如链表、树、栈等）运算符

    • 具有结构化的控制语句

      如 if … else 、 while语句、do … while语句、 switch语句、for语句
      用函数作为程序的模块单位，便于实现程序的模块化
      完全模块化结构化的语言

    • 语法限制不太严格，程序设计自由度大

      对数组下标越界不做检查
      对变量的类型使用比较灵活，例如，整型量与字符型数据可以通用
      放开了检查

    • 允许直接访问无力地址，能进行微操作，可以直接对硬件进行操作

      具有高级语言的功能和低级语言的许多功能，可用来编写系统软件
      双重性

    • 用C语言编写的程序可移植性好

      C的编译系统简介，很容易移植到新系统
      在新系统上远行时，可直接编译“标准链接库”中的大部分功能，不需要修改源代码
      几乎所有的计算机系统都可以使用C语言

1.4 最简单的 C语言程序

  • 例子
    例1.1 要求在屏幕上输出以下一行信息。
    This is a C program.

  • C语言允许用两种注释方式

    • //: 单行注释

      可单独一行
      可出现在一行种其他内容的右侧

    • / … /: 多行注释

例1.2 求两个整数之和。
例1.3 求两个整数中的较大者。

  • C语言程序的结构特点

    • 一个程序由一个或多个源程序文件组成
    • 小程序往往只包含一个源程序
    • 例1.1, 例1.2只有一个函数
    • 例1.3有两个函数
    • 一个源程序文件可以包含三个部分
    • 预处理指令
    • 全局声明
    • 函数定义
    • 函数是C程序的主要组成部分
    • 一个C程序是由一个或多个函数组成的
    • 必须包含一个main函数 (只能有一个)
    • 每个函数都用来实现一个或几个特定功能
    • 被调用的函数可以是库函数，也可以是自定义函数
    • 一个函数包括两个部分
    • 函数首部
      • 返回值类型
      • 函数名
      • 参数，若无参数，空括号或void
    • 函数体
      • 声明部分
        • 定义在本函数中所用到的变量
        • 对本函数所调用的函数进行声明
      • 执行部分：由若干个语句组成，指定在函数中所进行的操作
    • 程序总是从main函数开始执行
    • 程序中要求计算机完成的操作是由函数中的C语言完成的
    • C程序书写格式比较自由
      • 一行内可以些几个语句
      • 一个语句可以卸载多行上
    • 为清晰起见，习惯上每行只写一条语句
    • 数据声明和语句的最后必须有一个分号
    • C语言本身不提供输入输出语句
    • 程序应当包含注释，增加可读性

1.5 运行C程序的步骤与方法

  • 上机输入和编辑源程序
  • 对源程序进行编译
  • 进行连接处理
  • 运行可执行程序，得到运行结果

  • 对源程序进行编辑、编译和连接
  • 运行程序，分析结果

计算机网络概论

发表于 2018-04-01 | 分类于 网络与通信

1.1 计算机通信与网络发展过程

计算机网络在信息时代的作用

  • 计算机网络是计算机技术和通信技术相互结合的产物，是信息化社会的主要基础设施。
  • 计算机网络的作用是利用快速的信息传送，实现广泛的资源共享。
  • 计算机网络的发展中计算机技术和通信技术是相互制约，又相互促进发展的。
  • 计算机技术的发展以及计算机技术在通信系统中的应用，使得高速通信成为可能。高速通信的实现，又进一步促进计算机硬件、软件技术的发展。

计算机通信与网络的发展阶段

  • 面向终端的计算机网络
  • 多个计算机互连的计算机网络
  • 面向标准化的计算机网络
  • 面向全球互联的计算机网络

1.1.1 面向终端的计算机网络

  • 数量少
  • 终端作为从属设备
  • 近距离使用计算机终端

特点

  • 面向终端的计算机通信网
  • 集中式处理
  • 主从模式
  • 多个终端与主机通信
  • 数据传输速率较低
  • 一致性好
  • 通信开销较大，通信线路利用率低，对主机系统依赖性较大
  • 终端数量少，且与主机距离近
  • 计算机使用者大量时间用于交通。

1.1.2 多个计算机互连的计算机网络

分组交换的产生

  • ARPA(Advanced Research Project Agency)
  • 传统电路交换过于依赖某一条链路

分组交换网络的基本要求

  • 网络用语计算机之间的数据传送，而不是为了打电话。
  • 网络能够连接不同类型的计算机，不局限于单一类型的计算机。
  • 计算机在进行通信时，必须有冗余的路由。
  • 网络的结构应当尽可能地简单，同时还能够非常可靠地传送数据。

面向终端计算机系统与分组交换网的区别

  1. 早期的是以单个主机为中心的星状网
    • 共享
  2. 分组交换网是以网络为中心，主机都处在网络的外围。
    • 用户通过分组交换网可共享连接的

分组交换网特点

  • 独立工作的计算机系统
  • 允许异种机入网
  • 资源共享
  • 分散/分布控制
  • 分组交换
  • 专用的通信控制
  • 网络协议

因特网的发展

  • 三个阶段演进

  • 演进是逐渐的而不是突然的

    第一阶段

    • ARPANET
    • TCP/IP协议
    • 分解
    • Internet
    • ARPANET关闭

    第二阶段

    • 主干网
    • 地区网
    • 校园网

    第三阶段

    • ISP
    • NAP

1.1.3 面向标准化的计算机网络

网络体系结构标准化

  • 开放系统
  • 异种独立工作的计算机系统入网互联
  • 网络资源/用户资源共享
  • 层次结构和通信协议
  • 接口标准化
    • 国际标准化
    • ISO
      • ISO7498
      • ISO/OSI-RM国际标准

我国的网络发展现状

  • 低速数据网
  • CNPAC
  • CHINAPAC
  • CHINANET

1.2计算机网络基本概念

1.2.1 计算机网络的定义

  • 两台或两台以上的计算机相互连接起来才能构成网络，达到资源共享的目的。
  • 两台或两台以上的计算机连接，互相通信交换信息，需要一条通道。

  • 计算机之间要通信需要遵循一定的协议规则

    把分布在不同地点且具有独立功能的多个计算机，通过通信设备和线路连接起来，在功能完善的网络软件运行环境下，以实现网络中资源共享为目标的系统。

计算机网络与终端时分系统

  • 多用户系统

    • 多道批处理操作系统
    • 分时操作系统
    • 实时操作系统

      共享性：各终端用户共享主机资源
      并行性：各终端用户在一段时间内并行，同一时刻有多余两个或两个以上的用户都在运行。

计算机网络与多级系统

  • 多级系统

    • 数据流机
    • 多处理机
    • LAN
    • WAN

计算机网络与分布式系统

  • 分布式系统

    • 计算机硬件连接
    • 系统拓扑结构
    • 通信控制

  • 计算机网络

    • 分布式系统在分布式计算机操作系统支持下进行分布数据库处理和歌计算机之间的并行工作。
    • 分布式系统在计算机网络基础上为用户提供了同名的集成应用环境。
    • 分布式系统和计算机网络之间的区别：

      在软件上，而不是硬件
      移动文件在于由谁引发：系统或用户

1.2.2 计算机网络的组成

逻辑上可分成两部分：

  • 通信子网 网络信息的传输和交换
  • 终端系统 负责信息的处理。又称为资源子网
    • 网络软件

      网络管理及网络应用软件
      网络操作系统
      通信程序
      网络协议和协议软件

    • 网络系统的逻辑结构
    • 通信子网
    • 终端系统

1.3 网络的类型及其特征

  • 按网络的作用范围划分：LAN、MAN、WAN、AN
  • 按通信介质划分：有线网、无线网
  • 按通信传播方式划分：点到点、广播、多播
  • 按通信速率划分：低速(300bps-1.544Mbps)、快速(1.544Mbps-45Mbps)、高速(45Mbps-1000Mbps)
  • 按网络使用者划分：公用网、专用网
  • 按网络交换功能划分：电路交换、报文交换、分组交换、混合交换
  • 按网络控制方式划分：集中式、分布式
  • 按网络环境划分：部门网、企业网、校园网
  • 按网络拓扑结构划分：星形结构、层次结构或树形结构、总线型结构、环形结构

1.4 计算机通信协议与网络体系结构

  • 通信协议与分层体系结构
    a. 计算机通信是一个复杂的过程，想画通信的两个计算机系统必须高度协调工作才行，而这种“协调”是相当复杂的。
    b. “分层”可以将庞大而复杂的问题，转化为若干较小的局部问题，而这些较小的局部问题就比较易于研究和处理。
    c. 网络协议定义：通信双方必须遵循的、控制信息交换的规则的集合，是一套语义和语法规则，用来规定有关功能部件在通信过程中的操作，它定义了数据发送和接收工作中必经的过程。协议规定了网络中使用的格式、定时方式、顺序和检错。

    i. 语法：数据与控制信息的结构或格式，确定通信时采用的数据格式，编码及信号电平等。
ii. 语义：协议的语义指对构成协议的协议元素含义的解释
iii. 同步：规定了时间的执行顺序

    d. 网络协议的特点

    i. 层次性、可靠性和有效性。
ii. 协议的分层可以将复杂的问题简单化
iii. 协议可靠性和有效性是正常和正确通信的保证。

    e. 协议层次模型

    i. (n+1)层宇n层协议接口，n层提供服务
ii. n层宇(n-1)层协议接口，(n-1)层提供服务
iii. 实体(Entity)：是通信时能发送和接受信息的任何软硬件设施
iv. 接口(Interface)：是指网络分层结构中各相邻层之间的通信接口

    f. 分层的好处

    i. 各层之间是独立的
ii. 灵活性好
iii. 结构上可分割开
iv. 易于实现和维护
v. 能促进标准化工作

    g. 层数多少要适当

    i. 若曾数太少，就会使每一层的协议太复杂
ii. 层数太多又会在描述和综合各层功能的系统工程任务时遇到较多的困难。
iii. 设置合理的层数，有利于描述和综合这些层次功能。

    h. 分层的原则

    i. 按照功能分册、归类
ii. 层与层的接口适合于标准化
iii. 每一层只与相邻层又便捷
iv. 为满足各种通信服务需要，在一层内可形成若干子层

  • OSI-RM体系结构(开放系统互连)
    1-3数据通信 5-7信息处理

  1. 物理层
  2. 数据链路层
  3. 网络层
  4. 传输层
  5. 会话层
  6. 表示层
  7. 应用层
